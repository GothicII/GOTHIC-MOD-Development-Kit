Zum Index

Jedes GothicMOD wird durch eine INI-Datei "modname.ini" repräsentiert. Auch diese INI-Datei befindet sich im System-Verzeichnis von Gothic! Dadurch können nun mehrere Gothicspiele parallel installiert sein. Der GothicStarter verwendet diese, um eine Auswahl aller GothicMODs anzuzeigen.

Die INFO-Sektion

 
[INFO]
title=FunnyMod
description=Einige lustige Ergaenzungen zu Gothic
authors=M.Mustermann
version=0.3
webpage=http:://www.musterseite.de

In der INFO-Sektion sind allgemeine Informationen zu dem jeweiligen Spiel(MOD) abgelegt. Titel, Beschreibung, Autoren, Versionsnummer und eventuelle Webseite können hier eingetragen werden.

Die FILES-Sektion

 
[FILES]
; Dont attach file-extensions to these files. 
game=content\gothic
fightai=content\fight
menu=system\menu
camera=system\camera
music=system\music
soundeffects=system\sfx
particleseffects=system\particlefx
visualeffects=system\visualfx
outputunits=content\outputunits\ou
; List of vdf-files which have been added to the original gothic
; [separate them by spaces, include their extension ".mod"]
vdf=FunnyMod.mod

Die FILES-Sektion enthält die wichtigsten Daten. Hier sind alle Script-Dateien im Verzeichnis [GOTHIC]/_work/data/scripts aufgeführt, die möglicherweise verändert werden. Alle Pfadangaben sind relativ zum Scriptverzeichnis angegeben. Es ist sehr wichtig, dass eine Dateiendung nicht erlaubt ist. Diese wird vom Programm automatisch hinzugefügt.

Kandidat Nummer 1 für eine Anpassung ist der Wert fuer "game". Diese Datei enthält so ziemlich alle Scripte aus dem content-Ordner. Dabei erkennt Gothic automatisch, ob die Datei system/gothic.src benutzt werden muss (und damit neu compiliert wird), oder bereits eine aktuelle (compilierte) gothic.dat im Order "_compiled" vorliegt.

Der nächste Kandidat ist der Eintrag "outputunits". Dies sind in unserem Fall die beiden Dateien "ou.bin" und "ou.csl".

Alle anderen Dateien sind nicht so interessant.

Der Eintrag fuer "vdf" muss alle VDF-Dateien aufführen, die zusätlich zu denen von GOTHIC verwendet werden. Wenn dies mehrere sind, werden sie durch Leerzeichen getrennt aufgeführt (FunnyMod1.mod FunnyMod2.mod ...). Wir empfehlen aber, nur eine VDF-Datei zu verwenden, siehe dazu auch "Wie erstelle ich ein GothicMOD".

Die SETTINGS-Sektion

 
[SETTINGS]
world=test.zen
player=PC_HERO

Der world-Parameter gibt den bei "NEW GAME" zu ladenden Level an (relativ zum Verzeichnis _work/data/worlds/). Die Variable "player" zeigt auf eine NSC-Instanz, die als Spielercharakter (SC) verwendet wird. In unserem Fall also unser allseits bekannter "PC_HERO". Diego beispielsweise waere PC_THIEF, der Ricelord ware BAU_900_RICELORD. So kann man also selbst "geschusterte" Spielercharakter spezifizieren oder vielleicht gar Mud?! :-)

Die OPTIONS-Sektion

 
[OPTIONS]
force_parameters=
force_subtitles=0
show_info=0

Die letzte Sektion enthält diverse weitere Optionen:

"force_parameters" enthält Kommandozeilenparameter, die beim Start von Gothic automatisch verwendet werden; ohne, dass man sie angeben muss. Vielleicht moechte man ja, dass keine Musik gespielt wird ("force_parameters=-znomusic").

"force_subtitles" sollte von der Bedeutung klar sein. Ist der Wert auf 1 gesetzt wird die Einstellung in den Menueeinstellungen überschrieben, so dass Untertitel in jedem Fall ausgegeben werden. Sinnvoll, wenn keine akustische Sprachausgabe vorliegt.

"show_info" zeigt Titel und Versionsnummer im Spielmenü an, wenn der Wert auf 1 gesetzt wird.
